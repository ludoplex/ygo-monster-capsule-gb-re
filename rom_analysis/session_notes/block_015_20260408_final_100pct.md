## Methodology

### Professional Multi-Signal Block Classifier

Built `block_classifier.py` — a comprehensive static analysis tool using 5 independent
detection methods, each producing a 0.0-1.0 confidence score:

1. **Z80 Opcode Frequency Analysis**
   - Maintained a complete set of valid Game Boy CPU opcodes (the GB CPU is a
     modified Z80 that removes some instructions like IX/IY register ops)
   - Walked each block as a stream of instructions, tracking valid/invalid ratio
   - Bonus signals: RET (0xC9) count for function boundaries, CALL (0xCD) count
     for structured code, JR relative jump density for tight loops
   - Invalid opcodes (0xD3, 0xDB, 0xDD, etc.) penalized 3x

2. **Shannon Entropy Measurement**
   - Empty/padding: <1.0 bits/byte
   - Text (custom encoding): 3.0-5.5 bits/byte
   - Code: 5.0-6.5 bits/byte
   - Structured data: 4.0-6.0 bits/byte
   - Graphics: 2.0-6.0 bits/byte
   - Used as tie-breaker and confidence adjuster

3. **Game-Specific Text Encoding Detection**
   - Custom character table: a=0x00..z=0x19, A=0x64..Z=0x7D, 0=0x50..9=0x59
   - String terminators: 0xFF, 0x80, 0xFE
   - Control codes: 0x80-0x8F range
   - Dialogue vs names_list sub-classification based on inter-terminator spacing

4. **2bpp Tile Graphics Pattern Recognition**
   - GBC tiles: 8x8 pixels, 2 bits per pixel, 16 bytes per tile
   - Checked byte-pair correlation within tile rows
   - Zero-row detection (transparency)
   - Bit-plane overlap analysis

5. **Data Table Structure Detection**
   - Fixed-stride record scanning (tested 2/4/6/8/10/12/14/16/20-byte strides)
   - Column value range analysis (constrained columns indicate structured data)
   - Pointer table detection (16-bit LE values in 0x0000-0x7FFF range)

### Sub-type Classification

Each primary category gets a sub-type for richer documentation:
- **code**: `executable` (normal code) or `dispatch_table` (jump tables)
- **text**: `dialogue` (long strings) or `names_list` (short terminated strings)
- **data**: `structured_data` (fixed-stride records) or `pointer_table` (16-bit address arrays)
- **graphics**: `2bpp_tiles`
- **empty**: `padding_0xNN` (identifies fill byte)
