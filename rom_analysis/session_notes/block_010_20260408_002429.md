## Save Checksum — Alternative Theory

  The header bytes 0x0002-0x0003 may NOT be a checksum.
  Evidence:
    - No standard algorithm matches
    - Two different saves have very different values (0x9D5E vs 0xD056)
    - The game's save code reads A002/A003 but may just check for non-zero

  New theory: bytes 2-3 are a SAVE COUNTER or WRITE TIMESTAMP
  that increments each time the game saves. The game just checks
  if they're non-zero (meaning save exists) vs all-zero (no save).

  To test: play the game, save multiple times, compare bytes 2-3
  If they change each save → counter theory confirmed
  If constant → magic number theory

  For save editing: just preserve bytes 0-3 from a known-good save
  and only modify 0x4000+ roster data
